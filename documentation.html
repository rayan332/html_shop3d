Firebaze (Model3DLoader_Activity.kt)

You need to create a Firebase account and upload the .glb files for the 3D models. It is recommended to compress the .glb 3D models before uploading them, as this will optimize the experience.

Github (DownloadImage.kt)

the image file names should match the .glb file names, for example, char1.jpg and char1.glb .You need to create a GitHub account and upload a zip file named image.zip, which contains 3D model images at recommandation a resolution of 512x512.

Link of buttan (Model3DLoader_Activity.kt)

The green button is certainly meant to mislead the user on a website.

Json file (assets/animation.json)

This JSON file contains the name of the navigation image, for example 'character', and the names of the 3D models. It is structured in this format: 'character' : [[model1, animation1, animation2, ..., animation N]]. If there are no animations, the animation button is not displayed, and only the model is shown.

res/drawable (the navigation image)

These are the navigation images between models, for example, gun, moto, car etc. You can add as many as you want, but they need to be defined in animation.json

Start APP (MainActivity.kt)

You can modify the start screen text and image of the application. This image is defined in the drawable folder (loading.jpg) and will be displayed the first time. To update it, download the images.zip from GitHub.

the dialog box (MytopAppBar.kt)

You can modify the dialog box text here.

the dialog box (Slider_Lumier.kt)

You can modify the dialog box text here.

Be careful, if this value is 0f, it won’t be visible during development because its scale is 0f. It will gradually increase when the application starts. If you want to see it during development, set it to 1f just for testing.

Top Bar Application (MytopAppBar.kt)

The info icon displays the dialog box, and the number that increases represents the number of 3D models in the application. The shopping icon links to a website.
